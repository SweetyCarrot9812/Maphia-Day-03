# PACKAGES.md - Hanoa 생태계 패키지 가이드

Hanoa 생태계의 모든 패키지들에 대한 통합 가이드입니다.

## 📦 패키지 개요

Hanoa 생태계는 교육 및 생활 영역을 다루는 **개별 앱 생태계**로 구성됩니다.

### 🎯 현재 개발 중인 패키지들 (4개)

| 패키지 | 상태 | 플랫폼 | 설명 |
|--------|------|--------|------|
| **Clintest** | Phase 6 완료 | Desktop + Mobile | 의사/간호사 의학 학습 |
| **Lingumo** | 개발 중 | Mobile + Desktop | AI 언어 학습 플랫폼 |
| **AreumFit** | 완료 | Mobile + Backend | AI 피트니스 코칭 |
| **HaneulTone** | 초기 구조 | Multi-platform | AI 보컬 트레이닝 |

### 🚀 향후 개발 예정
- 기독교 관련 앱: 신앙 생활 지원
- 개인 비서 앱: AI 개인 어시스턴트  
- 학원 앱: 교육 기관 관리

## 🏥 Clintest - 의학/간호학 패키지

### 프로젝트 개요
- **목표**: 의사와 간호사를 위한 종합 의학 학습 플랫폼
- **현재 상태**: Phase 6 개발 완료 (Desktop v1.6.0 + Flutter v3.0)
- **과금 모델**: 단일 앱 구독

### 핵심 철학
> "AI는 분류·태깅·품질관리를 자동화하고, 사람은 콘텐츠 제공에만 집중"

### 주요 기능
- **AI 우선 설계**: GPT-5 Standard 기반 자동 분류
- **로컬 우선**: 100% 오프라인 지원, 사용자 데이터 완전 소유권
- **Mobile-Desktop 통합**: Flutter 앱 ↔ Desktop 서버 완전 연동
- **MongoDB Atlas 동기화**: 실시간 모바일-서버 데이터 동기화

### 최신 개발 현황 (2025.09)
- ✅ Conversational AI Setup System 완성
- ✅ 직종별 세분화 (간호사/의사/전문간호사)
- ✅ MongoDB 데이터베이스 통합 최적화
- ✅ Windows Electron 실행파일 배포

### 기술 스택
- **Desktop**: Node.js + Electron + MongoDB Atlas
- **Mobile**: Flutter + JWT 인증 + API 연동
- **AI**: GPT-5 Standard + 자동 태깅 시스템

## 🌐 Lingumo - 언어 학습 패키지

### 프로젝트 개요
- **목표**: AI 기반 언어 학습 플랫폼 (초기 영어, 이후 다국어)
- **대상**: 한국인 + 비한국인
- **현재 상태**: 개발 진행 중 (2025-09-04)

### 학습 플로우
1. **단어 입력**: 관리자 등록 기반
2. **예문 처리**: 관리자 예문 우선 → AI 자동 생성
3. **상황태그**: 사용자 맞춤 상황별 예문 생성
4. **미디어 연동**: YouTube/Netflix/뉴스 클립 자동 매칭
5. **스피킹 훈련**: 필수 발음 연습
6. **상황극/라이팅**: 선택형 고급 기능

### 난이도 시스템
#### 연령별 기준
- 유치원: 2-6 단어
- 초등학교: 4-15 단어  
- 중학교: 10-20 단어
- 고등학교: 15-30 단어

#### 성인 CEFR 기준
- A1: 6-10 단어
- A2: 8-12 단어
- B1: 10-15 단어
- B2: 12-18 단어
- C1: 15-25 단어
- C2: 20-30 단어

### 기술 스택
- **플랫폼**: Flutter Mobile + Electron Desktop
- **AI**: 예문 자동 생성 + 발음 평가
- **미디어**: YouTube/Netflix API 연동

## 💪 AreumFit - 피트니스 패키지

### 프로젝트 개요
- **목표**: AI 기반 개인 헬스/크로스핏 코칭 시스템
- **대상**: Hanoa 사용자 (개인용)
- **완료일**: 2025-09-04
- **특징**: GPT-5 Standard AI 코치, 점진적 과부하, SRS 운동량 관리

### AI 코치 시스템
#### 핵심 AI 모델
- **메인**: GPT-5 Standard (`gpt-5-standard`)
- **예비**: GPT-4o (fallback)
- **API**: `/ai/generate` 엔드포인트

#### 주요 기능
1. **오늘의 운동 계획**: 개인 기록 분석 후 근육군 자동 선택
2. **운동별 세부 추천**: 
   - 중량(kg), 반복(회), 세트, 휴식(초)
   - RPE 예측 및 점진적 과부하 적용
3. **실시간 대화**: 자연어 운동 상담
4. **개인화 추천**: 최근 2주 기록 기반 분석

### 기술 스택
```
areumfit/                    # Flutter 앱
├── lib/db/schema.dart      # Drift 데이터베이스
├── services/ai_coach_service.dart  # GPT-5 통합
└── models/                 # 데이터 모델

areumfit-backend/           # Node.js 백엔드 (포트 3004)
├── src/routes/ai.js       # AI 제안 처리
├── auth/auth.js           # JWT 인증
└── models/mongodb.js      # MongoDB 스키마
```

## 🎵 HaneulTone - 보컬 트레이닝 패키지

### 프로젝트 개요
- **목표**: 전문 보컬리스트와 일반인을 위한 AI 보컬 트레이닝
- **현재 상태**: 초기 Flutter 프로젝트 구조 생성 완료 (2025.09.03)
- **과금 모델**: 프리미엄 보컬 코칭 서비스

### 핵심 철학
> "모든 사람은 아름다운 목소리를 가지고 있습니다. AI 기술로 개인별 음성 특성을 분석하고, 체계적인 보컬 트레이닝을 제공합니다."

### 기술 특징
- **AI 우선 설계**: 실시간 음성 분석 및 보컬 테크닉 피드백
- **개인화 학습**: 음역대 분석 및 맞춤형 트레이닝 프로그램
- **멀티플랫폼**: Android, iOS, Web, macOS, Windows, Linux (6개 플랫폼)

### 개발 현황 (2025.09.03)
- ✅ 완전한 Flutter 프로젝트 구조 생성 (8.2MB)
- ✅ 6개 플랫폼 지원 준비 완료
- ✅ 권한 설정 (마이크 접근) 준비
- ✅ PWA 지원 및 오프라인 기능 준비

### 데이터 구조
```
data/
├── audio_references/     # 표준 발음 참조 오디오
├── pitch_data/          # 성조 데이터 저장소
├── sessions/            # 학습 세션 기록
└── user_recordings/     # 사용자 녹음 파일
```

## 🔧 공통 기술 아키텍처

### 로컬 우선 + Firebase 동기화
- **기본 저장**: 로컬 DB (Isar) 우선
- **동기화**: Firebase 효율적 연동
- **방식**: 완전 오프라인이 아닌 하이브리드

### Hanoa 멀티모델 라우팅 시스템
```
메인 비서: OpenAI GPT-5 Standard (톤·안전·최종 응답)

팀원 역할:
- GPT-5 하위모델: 짧은 요약·태깅·간단 퀴즈
- Gemini: 이미지/표/아이디어 발산  
- Claude: 장문 편집·정책·톤 교정

운영 모드: 균형(balanced)
상담사 바이어스: ON, 강도 +0.10
```

### 배포 전략
- **현재**: 각 앱 개별 배포
- **장기**: 패키지 기반 통합 구독 모델 검토
