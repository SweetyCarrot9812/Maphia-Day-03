## 1. 제품 개요

### 배경 & 목적
포트폴리오 프로젝트로서 웹 애플리케이션 개발 역량을 증명하기 위한 회의실 예약 시스템 구축. 실제 업무 환경에서 사용되는 예약 시스템의 핵심 기능들을 구현하여 풀스택 개발 능력을 보여준다.

### 타겟 사용자
- **Primary**: 포트폴리오 리뷰어 (채용 담당자, 개발팀)
- **Secondary**: 실제 사용 시뮬레이션을 위한 테스트 사용자

### 성공 지표
| 지표 | 베이스라인 | 목표 | 측정 방법 |
|------|------------|------|-----------|
| 기능 완성도 | 0% | 100% | 통과조건 A-F 모두 구현 |
| 페이지 로딩 속도 | - | <3초 | Vercel Analytics |
| 예약 플로우 완료율 | - | >90% | GA4 Funnel |
| 코드 품질 | - | Clean Code | ESLint + 주석 |

### 범위
- ✅ **In Scope**:
  - 회의실 목록 조회 및 선택
  - 시간대별 예약 현황 확인
  - 예약 등록 (예약자 정보 + 비밀번호)
  - 예약 조회 (휴대폰번호 + 비밀번호)
  - 어드민 패널 (회의실 관리)
  - Vercel 배포 + Supabase DB 연동

- ❌ **Out of Scope**:
  - 실제 결제 시스템
  - 이메일/SMS 알림
  - 복잡한 권한 관리
  - 다국어 지원
  - 모바일 앱

## 3. 사용자 여정

### **일반 사용자 (예약)**:
1. 메인 페이지 → 2. 회의실 선택 → 3. 날짜/시간 선택 → 4. 예약 정보 입력 → 5. 예약 완료

### **일반 사용자 (조회)**:
1. 메인 페이지 → 2. "내 예약 조회" → 3. 휴대폰번호+비밀번호 입력 → 4. 예약 정보 확인

### **관리자**:
1. Admin 로그인 → 2. 대시보드 → 3. 회의실 관리 또는 예약 현황 조회

## 4. 기술 요구사항

### 기술 스택
- **Frontend**: Next.js 14 + TypeScript + Tailwind CSS
- **Backend**: Next.js API Routes + Supabase
- **Database**: PostgreSQL (Supabase)
- **배포**: Vercel
- **인증**: Supabase Auth (Admin만)

### 성능 SLO
- 페이지 로딩: p95 < 3초
- API 응답: p95 < 500ms
- Lighthouse Score: >90

### 데이터베이스 스키마
```sql
-- 회의실 테이블
conference_rooms (
  id, name, location, capacity,
  created_at, updated_at
)

-- 예약 테이블
bookings (
  id, room_id, user_name, phone,
  purpose, password_hash, booking_date,
  start_time, end_time, created_at
)
```

## 5. 실험 설계

### A/B 테스트 계획
- **대상**: 예약 플로우 UX
- **가설**: 단일 페이지 vs 스텝 방식 중 어느 것이 완료율이 높은가
- **샘플**: 최소 100명 (포트폴리오 리뷰어 + 테스트 사용자)
- **기간**: 2주
- **측정**: 예약 완료율, 이탈 지점 분석

### Go/No-Go 기준
- 모든 통과조건 구현: Must
- 주요 버그 0개: Must
- 페이지 로딩 <3초: Must
- 반응형 동작: Should
