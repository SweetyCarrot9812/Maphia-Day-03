## 1. 핵심 사용자 여정 맵

### 주요 사용자 목표
1. **탐색**: 포트폴리오 사이트에서 쇼핑몰 프로젝트 체험
2. **체험**: 실무 수준의 쇼핑 기능 테스트
3. **평가**: 개발자의 기술 역량 확인

### 사용자 여정 단계
```
[포트폴리오 사이트] → [프로젝트 링크] → [쇼핑몰 랜딩] → [상품 탐색]
     ↓
[상품 선택/해제] → [수량 조절] → [가격 계산 확인] → [반응형 테스트] → [완료]
```

### 감정 여정 (Emotion Journey)
- **호기심** (포트폴리오 발견)
- **기대감** (프로젝트 클릭)
- **탐색** (기능 테스트)
- **만족/실망** (품질 평가)
- **의사결정** (채용/추천 판단)

```

## 6. 접근성 (A11y) 플로우

### 6.1 키보드 내비게이션 플로우
```
Keyboard Navigation Flow
│
├─ Tab 순서
│   │
│   ├─ 1. 전체 선택 체크박스
│   ├─ 2. 상품1 체크박스
│   ├─ 3. 상품1 수량 감소 버튼
│   ├─ 4. 상품1 수량 증가 버튼
│   ├─ 5. 상품2 체크박스
│   └─ ... (반복)
│
├─ 스크린 리더 플로우
│   │
│   ├─ "상품 선택됨, 체크박스"
│   ├─ "수량 3개, 스핀버튼"
│   ├─ "가격 15,000원"
│   └─ "배송비 무료"
│
└─ Focus 관리
    │
    ├─ 체크박스 선택 시 → 수량 버튼으로 포커스 이동
    ├─ 수량 변경 시 → 다음 상품으로 포커스 유지
    └─ 에러 발생 시 → 관련 입력 필드로 포커스 이동
```

### 6.2 ARIA 라벨링 플로우
```
ARIA Labeling Flow
│
├─ 동적 콘텐츠
│   │
│   ├─ aria-live="polite" → 가격 업데이트 알림
│   ├─ aria-live="assertive" → 에러 메시지 즉시 알림
│   └─ aria-describedby → 배송비 안내 연결
│
├─ 상호작용 요소
│   │
│   ├─ aria-label="상품 선택" → 체크박스
│   ├─ aria-label="수량 증가" → + 버튼
│   ├─ aria-label="수량 감소" → - 버튼
│   └─ aria-valuemin/max → 수량 범위
│
└─ 그룹화
    │
    ├─ role="group" → 각 상품 영역
    ├─ aria-labelledby → 상품명과 가격 연결
    └─ fieldset/legend → 전체 선택 영역
```

## 8. 테스트 사용자 시나리오

### 8.1 기본 체험 시나리오 (2분)
```
Basic Experience Test
│
├─ 목표: 기본 쇼핑 기능 체험
│
├─ 시나리오:
│   │
│   ├─ 1. 사이트 접속 (0-10초)
│   ├─ 2. 상품 3개 선택 (10-30초)
│   ├─ 3. 수량을 총 5개로 조정 (30-50초)
│   ├─ 4. 배송비 규칙 확인 (50-70초)
│   ├─ 5. 전체 선택/해제 테스트 (70-90초)
│   └─ 6. 최종 총액 확인 (90-120초)
│
└─ 측정 지표:
    │
    ├─ 작업 완료율: 100%
    ├─ 평균 완료시간: <120초
    ├─ 오류 횟수: 0회
    └─ 사용자 만족도: 4점 이상/5점
```

### 8.2 반응형 테스트 시나리오 (1분)
```
Responsive Test Scenario
│
├─ 목표: 다양한 화면에서 일관된 경험
│
├─ 시나리오:
│   │
│   ├─ 1. 데스크톱에서 상품 2개 선택 (0-20초)
│   ├─ 2. 브라우저 창 크기를 모바일로 축소 (20-30초)
│   ├─ 3. 선택 상태가 유지되는지 확인 (30-40초)
│   ├─ 4. 모바일에서 수량 조절 테스트 (40-50초)
│   └─ 5. 다시 데스크톱으로 확대 (50-60초)
│
└─ 측정 지표:
    │
    ├─ 상태 유지율: 100%
    ├─ 레이아웃 깨짐: 0건
    ├─ 터치 영역 정확도: >90%
