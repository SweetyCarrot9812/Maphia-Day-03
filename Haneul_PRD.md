# Haneul PRD (Product Requirement Document)

## 1. 제품 개요

### 한 줄 요약
개인 맞춤형 AI 에이전트가 Obsidian에 아이디어와 할 일을 정리하고, 우선순위를 점수화(시급성+중요도)해 추천하며, 이메일과 캘린더로 알림을 보내주는 시스템.

### 핵심 가치
아이디어 과부하를 줄이고, 체계적인 우선순위 관리와 프로젝트 현황을 한눈에 확인.

### 주요 사용자
다수의 프로젝트를 동시에 진행하는 개인(예: 창업가, 개발자, 연구자).

## 2. 기능 요구사항 (MoSCoW 방식)

### 🟢 Must (필수)

#### Obsidian 연동
- 아이디어와 할 일을 자동으로 기록/정리
- 태그 및 폴더 기반 구조화
- **AC**: 사용자가 메모 추가 시 Obsidian 노트에 자동 반영
- **KPI**: 하루 기록 반영 성공률 ≥ 95%

#### 우선순위 점수화 (시급성+중요도)
- 각 아이디어에 대해 두 항목을 점수(1~10)로 매기고 합산
- **AC**: 사용자가 입력한 점수 기준으로 우선순위 정렬된 리스트 제공
- **KPI**: 사용자 피드백 기준 80% 이상 "적절하다" 응답

#### 이메일 알림 연동
- 하루 한 번 우선순위 TOP 3 알림 전송
- **AC**: 설정한 시간대에 알림이 정상 발송
- **KPI**: 발송 성공률 ≥ 98%

#### 캘린더 연동
- 우선순위 높은 항목 → 캘린더 일정 생성 제안
- **AC**: 선택 시 구글 캘린더에 일정 자동 등록
- **KPI**: 일정 생성 기능 사용률 ≥ 60%

#### 메모리 규칙 적용
- "영구 기억": 완료된 작업, 정리된 아이디어 기록
- "임시 기억": 세션 내 대화 기록
- **AC**: 세션 종료 후 임시 기억은 삭제, 영구 기억은 Obsidian에 저장
- **KPI**: 데이터 손실률 ≤ 2%

### 🟡 Should (권장)
- 프로젝트별 현황 대시보드 (진행 상황 요약)
- 태그 기반 자동 분류 (예: #개발, #아이디어, #개인)

### 🔵 Could (선택)
- 다국어 입력/출력 지원
- 음성 입력 및 음성 알림 기능

## 3. 가드레일

### 개인정보
모든 데이터는 로컬 우선 저장, 클라우드 동기화 시 사용자 동의 필수.

### 민감 정보
의료·재정 조언은 정보 제공용으로 한정, 전문 상담 권고 문구 자동 첨부.

### 보안
이메일·캘린더 연동은 OAuth 인증 기반.

### 저작권
Obsidian 노트·캘린더 데이터는 사용자의 소유로 명확히 고지.

## 4. 질문 (누락 정보 보완)

1. Obsidian → 이메일/캘린더로 데이터 전송 시 플랫폼·API 선택은 무엇으로 할까요?
2. 알림 빈도는 하루 1회로 확정할까요, 아니면 사용자 설정 가능하게 할까요?
3. 우선순위 점수(시급성+중요도)는 사용자가 직접 입력할까요, AI가 추론할까요?
4. 프로젝트 대시보드 뷰는 필수 기능(Must)로 포함할까요, 권장(Should)으로 둘까요?
5. 초기 배포 플랫폼은 PC/웹 기반일까요, 모바일 앱까지 확장할까요?

## 🚀 MVP 요약

### 핵심 목표
"Obsidian에 기록된 아이디어/할 일을 우선순위 점수화해 정리하고, 이메일+캘린더 알림으로 하루를 관리하는 AI 에이전트"

### MVP 스코프 (1주 내 완성)
1. Obsidian 기록 자동화
2. 우선순위 점수화(시급성+중요도)
3. 이메일 알림 (TOP 3)
4. 캘린더 일정 생성
5. 메모리 규칙 적용

### 👉 Hook
"Obsidian에 메모만 남겨도, 내 AI 에이전트가 알아서 '이걸 먼저 해보자!' 하고 정리해주고, 이메일·캘린더까지 챙겨준다. 이제 아이디어 과부하에서 벗어나볼까?"
